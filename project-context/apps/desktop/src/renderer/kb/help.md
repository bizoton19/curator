# Curator Help

## Baseline
Baseline is the project foundation. Use it to capture the scope, assumptions, and a short overview. It should answer: what are we estimating, what is in/out of scope, and what assumptions drive cost.

## Requirements
Requirements define the detailed goals and constraints for the estimate. Use this to list success criteria, constraints, dependencies, and inputs the estimator needs.

## Tasks
Tasks is the estimator’s execution plan. Use it to break down the analysis steps the agent should perform in sequence and to describe the desired outputs.

## Context Documents
Context documents provide evidence and domain knowledge (rate cards, prior estimates, architecture docs, contracts). Add them to `context-documents/` via drag‑drop or the + button.

## Templates & Export
Templates are `.docx` or `.dot` files used to format the final cost estimate. Upload templates in the `templates/` section, then generate a cost estimate and export to DOCX.

## Workspaces
Each workspace is a project folder with `baseline.md`, `requirements.md`, and `tasks.md`. The left panel shows all workspaces and their files. You can switch workspaces at any time.

## Search
Use the search box in the Explorer to find content across core files and supporting documents. Search hits open the file so you can review or edit.

## Chat & Copilot
Chat helps refine the estimate, answer questions, and apply edits. “Agent” mode can propose file edits for review. “Ask” mode gives advice without modifying files.

## Settings & API Keys
Settings lets you choose a model provider and enter an API key. Keys are stored securely in local encrypted storage.
